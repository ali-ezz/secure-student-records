# Contributing to SRMS

Thank you for your interest in contributing to Secure Student Records Management System.

## How to contribute

1. Open an issue to describe the bug or feature request.
2. Fork the repository and create a branch for your changes.
3. Keep your branch focused on a single change.
4. Add tests for any bug fixes or new behavior.
5. Submit a pull request with a clear title and description.

## Code style

- Keep changes small and focused.
- Use descriptive commit messages.
- Document behavior in `README.md` when needed.

## Reporting issues

For security issues, see `SECURITY.md`.
